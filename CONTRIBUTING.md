# Contributing

Dziękujemy za zainteresowanie projektem.

## Jak zacząć

1. Sforkuj repozytorium.
2. Utwórz branch z opisową nazwą.
3. Wprowadź zmiany wraz z testami.
4. Upewnij się, że dokumentacja jest aktualna.
5. Otwórz Pull Request.

## Zasady

- utrzymujemy czytelny kod i małe, zrozumiałe commity
- opisujemy zmiany w README lub `docs/`, gdy wpływają na sposób użycia
- nie commitujemy sekretów ani kluczy API
- testy powinny przechodzić lokalnie przed wysłaniem PR
