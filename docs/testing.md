# Strategia testów

## Poziomy testów

- testy jednostkowe dla logiki aplikacji
- testy integracyjne dla przepływów importu, zapisu i eksportu
- testy ręczne dla jakości rozpoznawania notatek
- testy releasowe dla paczki `.exe`

## Założenia

- odpowiedzi AI powinny być mockowane w większości testów automatycznych
- przykładowe dane testowe powinny obejmować różne style pisma i jakość zdjęć
- każda ważna funkcja powinna mieć przynajmniej podstawowy test
